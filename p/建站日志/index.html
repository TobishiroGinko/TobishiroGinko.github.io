建站日志~使用Hugo+Stack搭建起一个博客

Apr 13, 2024
阅读时长: 3 分钟

# 缘起

”记录思考是提高思考质量的关键“

因为先前并没有一个记录自己所学的习惯，在无数次看到一些似曾相识但是忘记了解决方法的问题后，痛定思痛，决定搭建一个自己的博客，用来记录自己的学习、生活。

# 使用Hugo+Stack搭建博客

# 为什么选择Hugo？

Hugo 是由 Go 语言实现的静态网站生成器。简单、易用、高效、易扩展、快速部署。

主流的博客框架有Hexo、WordPress、Hugo等等，最终，我选择了Hugo，原因如下

  • Hugo是一个静态博客框架，搭配Github Page，零成本搭建博客~
  • 快速生成，调试方便。在源文件上的修改基本都能实时渲染到网页上，非常方便。
  • 依靠Go语言进行开发，性能强大。近期有在研究Go，对于Hugo有天然滤镜。
  • 构建速度快，大量文章的情况下也能快速生成，适合长期使用。

# 安装Hugo

Hugo安装可以参考 官方文档中的安装方法
我的系统是Windows 11，采用文档中预构建的二进制文件方式安装，版本应为拓展版，即版本后有extended的版本。 下载完成后将其添加进环境变量Path中。 安装完成后，可以输入以下命令检查Hugo版本。

1

hugo version

若版本后带+extended，则版本安装成功。

# 创建博客

使用Hugo的第一步便是创建站点，参考官方文档，需要注意：

如果您是 Windows 用户：

  • 不要使用命令提示符
  • 不要使用 Windows PowerShell
  • 从 PowerShell 或 Linux 终端（如 WSL 或 Git Bash）运行这些命令

PowerShell 和 Windows PowerShell 是不同的应用程序。

使用Git Bash，执行以下命令新建站点。

1

hugo new site 站点目录

# 安装主题及使用

接下来我们进行主题安装，主题可以在Hugo Theme或Github找到。这里我选择的是Stack模板，简约大方现代化，符合我的审美。安装主题可以在themes文件夹中执行clone将主题仓库克隆到本地，也可以直接下载解压到themes文件夹，但要注意下载解压完成的文件夹需要修改文件夹名称，将后面的-master删除。这里我采用了clone方式进行安装。

1
2

cd themes/
git clone https://github.com/CaiJimmy/hugo-theme-stack.git

Stack主题内置了exampleSite，将文件夹内的文件复制到根目录，即可快速部署示例网站，接下来的步骤在示例网站的基础上进行自定义。修改好后，即可在根目录使用如下命令启动本地服务器。

1

hugo server

注意：此时国内用户大概率无法启动，需要删除/content/post/rich-content/index.md中的Youtube，Twitter，Video Simple的ShortCode，也可以直接删除/content/post内的全部文件
启动成功后就可以在浏览器访问localhost:1313来访问本地网页了。

<script> //正文字重300，标题字重700

            customFont.href = "https://fonts.googleapis.com/css2?family=Noto+Serif+SC:wght@300;700&display=swap";
            customFont.type = "text/css";
            customFont.rel = "stylesheet";

            document.head.appendChild(customFont);
        }());
</script>

我还修改了assets/scss/variable.scss文件，主题使用了 CSS Variablesopen 来定义字体，中文字体的定义在 --zh-font-family 这个变量里，具体就是调整了--zh-font-family和--base-font-family的顺序，代码如下：

主题中的背景、字体颜色都可以在assets/scss/variable.scss文件中修改，我只修改了字体颜色，让其与背景颜色更贴合，修改代码如下：

将你的图片转换为.ico文件放入/static文件夹中，修改hugo.yaml配置站点图标为你刚刚放入/static文件夹的图标，具体代码如下：
